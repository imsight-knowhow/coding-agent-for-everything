# Plans 目录

## 用途

记录实施策略、功能路线图和多步骤开发计划。将复杂功能分解为可管理的任务。

## 内容类型

- 功能实施路线图
- 系统迁移策略
- 版本升级计划
- 重构计划
- 技术债务清理计划

## 命名规范

使用功能或系统名称：
- `api-redesign-roadmap.md`
- `database-migration-strategy.md`
- `v2-implementation-plan.md`
- `testing-infrastructure-plan.md`

## 示例

```markdown
# API 重新设计路线图

## HEADER
- **Purpose**: 规划 API 从 v1 到 v2 的迁移路径
- **Status**: In Progress
- **Date**: 2025-01-15
- **Dependencies**: 当前 API 文档, 用户反馈
- **Target**: 开发团队和 AI 助手
```
