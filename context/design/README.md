# Design 目录

## 用途

存储 API 规范、架构图和技术设计文档。包括高层系统设计和详细组件规范。

## 内容类型

- API 规范文档
- 数据库架构设计
- 系统架构图
- 认证流程设计
- 组件交互设计

## 命名规范

使用组件或系统焦点命名：
- `api-specification.md`
- `database-schema-design.md`
- `authentication-flow.md`
- `system-architecture.md`

## 示例

```markdown
# API 规范

## HEADER
- **Purpose**: 定义 RESTful API 接口规范
- **Status**: Active
- **Date**: 2025-01-15
- **Dependencies**: 数据库架构
- **Target**: 后端开发者和 AI 助手
```
